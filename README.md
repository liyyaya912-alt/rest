# 智慧產線稼動率與異常排查看板

## 專案簡介

本專案是一個以 **Python Streamlit** 製作的工業工程（Industrial Engineering, IE）應用看板，主題為「智慧產線稼動率與異常排查」。

此作品模擬工廠現場的產線戰情室 Dashboard，讓操作員、基層生管或設備工程師可以快速輸入機台異常資訊，並即時查看產線時間稼動率、故障停機比例，以及對應的異常排查與復歸 SOP。

專案重點不只是單純顯示數據，而是將 **IE 稼動率分析、機台 Alarm 診斷、現場安全點檢、TPM 預防保養觀念** 整合到同一個互動式網頁介面中。

## 設計目標

本看板的設計目標如下：

1. 協助現場人員快速掌握機台異常狀態。
2. 將計劃運轉時間與故障停機時間轉換為時間稼動率指標。
3. 透過圓餅圖視覺化呈現實際運轉時間與停機時間比例。
4. 依據錯誤代碼輸出對應的異常根因分類與標準復歸流程。
5. 將 TPM 預防性維護觀念導入異常處置建議。
6. 建立符合工業工程師實務使用情境的網頁化看板。

## 使用情境

當產線機台發生異常停機時，現場人員可透過此看板進行初步判斷：

- 選擇故障機台型號
- 選擇錯誤代碼或 Alarm Code
- 輸入計劃運轉時間
- 輸入故障停機時間
- 查看時間稼動率與時間分配圖
- 依照系統提供的安全檢查與 SOP 進行復歸

此工具適合用於課堂作業展示、IE 改善提案雛形、設備管理概念展示，以及 Streamlit 資料應用練習。

## 系統功能

### 1. 參數選取與數據控制

左側 Sidebar 提供現場操作員可調整的輸入參數：

- 故障機台型號：
  - CNC-01
  - 封裝機-A
  - 點膠機-B
- 錯誤代碼 / Alarm Code：
  - Err-401 馬達過熱
  - Err-302 感應器異常
  - Err-105 實體卡料
- 計劃運轉時間：
  - 範圍 120 至 480 分鐘
  - 預設 480 分鐘
- 故障停機時間：
  - 範圍 0 至 120 分鐘
  - 預設 30 分鐘

### 2. 稼動率指標計算

系統會自動計算：

```text
實際運轉時間 = 計劃運轉時間 - 故障停機時間
時間稼動率 = 實際運轉時間 / 計劃運轉時間 × 100%
```

並以三個數據方框呈現：

- 計劃時間（min）
- 停機時間（min）
- 時間稼動率（%）

### 3. 圖表視覺化

使用 Plotly 繪製時間分配圓餅圖，清楚呈現：

- 實際運轉時間
- 故障停機時間

透過圖表可以快速判斷停機時間占比，方便 IE 人員進一步進行改善分析。

### 4. AI Skill 智慧診斷報告

系統會依據選取的 Alarm Code 產生對應的「機台異常排查與處置報告」，內容包含：

- 異常現況與初步分類
- 故障可能類別
- 異常摘要
- 現場安全引導檢查
- 標準排除復歸 SOP
- TPM 預防性維護建議

報告格式模擬設備工程師在現場進行 Troubleshooting 時的處理邏輯。

## 異常分類邏輯

目前系統內建三種常見製造業異常情境：

| 錯誤代碼 | 異常名稱 | 初步分類 | 常見原因 |
| --- | --- | --- | --- |
| Err-401 | 馬達過熱 | 電氣/控制問題 | 長時間高負載、散熱不良、風扇異常、軸承阻力過大 |
| Err-302 | 感應器異常 | 電氣/控制問題 | 感應器髒污、位置偏移、線路鬆脫、PLC I/O 訊號異常 |
| Err-105 | 實體卡料 | 機構/機械問題 | 輸送帶偏移、導軌間隙不當、料件變形、異物阻塞 |

## 本機執行方式

請先安裝套件：

```bash
pip install -r requirements.txt
```

啟動 Streamlit：

```bash
streamlit run app.py
```

啟動後在瀏覽器開啟：

```text
http://localhost:8501
```

## 部署到 Streamlit Cloud

若要將此專案部署到 Streamlit Cloud，可依照以下流程：

1. 將此資料夾上傳到 GitHub Repository。
2. 確認 Repository 中包含：
   - `app.py`
   - `requirements.txt`
   - `README.md`
   - `.gitignore`
3. 前往 Streamlit Cloud。
4. 選擇 GitHub Repository。
5. Main file path 設定為：

```text
app.py
```

6. 點選 Deploy 即可完成部署。

## 工業工程應用價值

此看板可作為 IE 現場改善分析的初步工具，協助管理者快速掌握：

- 設備停機對稼動率的影響
- 異常類型與停機時間的關聯
- 常見 Alarm 的處置流程
- 現場點檢項目的標準化
- TPM 自主保養項目的建立方向

若後續擴充資料庫或串接真實機台資料，也可進一步發展為更完整的設備監控系統。
